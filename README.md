## 📖 Descripción del Proyecto

Este proyecto consiste en el desarrollo de una **plataforma web completa para la venta y gestión de cursos**, inspirada inicialmente en la Escuela Vallenata Héctor Ibáñez.

La plataforma permite a los usuarios **explorar, comprar y realizar cursos en línea**, mientras que los profesores y administradores pueden **gestionar cursos, contenido y métricas desde un panel interno avanzado**.

Está construida como una aplicación moderna de tipo **e-learning (EdTech)**, combinando frontend, backend, base de datos y servicio de archivos en una sola solución coherente.

---

## 📱 Diseño Responsive

El proyecto está diseñado para funcionar correctamente en **diferentes tamaños de pantalla**, garantizando una buena experiencia en:

- 📱 **Dispositivos móviles**
- 💻 **Computadores de escritorio**
- 📟 **Tablets**

Se desarrolla siguiendo un enfoque **responsive-first**, asegurando que la interfaz sea clara, usable y visualmente atractiva en cualquier dispositivo.

---

## 🎯 Objetivo Final del Proyecto

El objetivo es crear una **plataforma profesional de cursos online** con múltiples funcionalidades avanzadas.

### Para estudiantes

- Registro e inicio de sesión
- Exploración del catálogo de cursos
- Compra de cursos
- Acceso al contenido del curso (módulos y lecciones)
- Seguimiento del progreso
- Panel personal de estudiante

### Para profesores / administradores

- Gestión completa de cursos (crear, editar, publicar, eliminar)
- Subida y organización de imágenes y videos
- Gestión de estudiantes
- Control de ventas e historial de transacciones
- Monitoreo de actividad de usuarios
- Dashboard con métricas y estadísticas

## 🧩 Funcionalidades Planeadas

### Sistema de usuarios

- Registro e inicio de sesión
- Autenticación segura
- Recuperación de contraseña

### Sistema de roles

La plataforma contará con un **sistema de roles avanzado**:

| Rol | Descripción |
|---|---|
| Administrador | Acceso total a la plataforma |
| Instructor | Gestión de sus propios cursos y contenido |
| Estudiante | Acceso a cursos comprados |

### Sistema de cursos

- Catálogo de cursos con interfaz pública
- Página individual de cada curso
- Contenido estructurado por módulos y lecciones
- Seguimiento del progreso del estudiante

### Sistema de compras

- Compra de cursos
- Registro de transacciones
- Historial de compras del estudiante
- Control de acceso al contenido comprado

### Dashboard administrativo

- Métricas de ventas
- Monitoreo de usuarios
- Control y publicación de cursos
- Estadísticas de actividad
- Gestión del contenido educativo

## 🛠 Tecnologías del Proyecto

### Frontend

| Tecnología | Uso |
|---|---|
| **Next.js** | Framework principal, rutas y páginas |
| **React** | Construcción de la interfaz |
| **TypeScript** | Tipado estático |
| **Tailwind CSS** | Estilos y diseño responsive |
| **shadcn/ui + Radix UI** | Componentes de interfaz reutilizables |
| **React Hook Form** | Manejo de formularios |
| **Zod** | Validación de datos |
| **TanStack Table** | Tablas con funcionalidades avanzadas |
| **Sonner** | Notificaciones tipo toast |
| **Lucide React** | Librería de iconos |
| **next-themes** | Soporte de temas y modo oscuro |
| **UploadThing** | Subida de archivos (imágenes y videos) |
| **Drag and Drop** | Interacciones de arrastrar y soltar |

### Backend

El proyecto utiliza las **rutas API nativas de Next.js** para construir el backend, manejando operaciones como:

- Crear, consultar, actualizar y eliminar cursos (CRUD)
- Gestión de la subida de imágenes y videos
- Lógica de negocio y control de acceso
- Autenticación y manejo de sesiones

Este enfoque permite entender profundamente el flujo completo de una aplicación web moderna sin depender de un servidor separado.
